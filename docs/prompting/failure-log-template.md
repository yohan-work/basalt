# 실패 로그 템플릿

각 실패는 다음 형식으로 기록해 다음 프롬프트 개선 루프로 연결합니다.

```md
## 실패 기록
- 날짜:
- 시나리오:
- 사용 템플릿/패밀리:
- 입력 요약:
- 기대 출력:
- 실제 출력:
- 실패 유형: (해석오류/제약위반/포맷오류/상태오류/성능저하)
- 근본원인 추정:
- 반영 수정안:
- 적용 여부: (적용/보류)
- 결과 재확인: (pass/fail)
```

정책
- 동일 패턴은 3회 이상 누적 시 템플릿을 새 버전으로 분기
- 실패가 상태 제약 미준수라면 API 계약 문서(`api-contracts.md`)를 먼저 업데이트 후 재실행
