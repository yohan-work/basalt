# Agent/Skill 계약서 모음

에이전트와 스킬 호출의 의도 충돌을 줄이기 위해 목적별 계약을 분리합니다.

## Agent Contract: software-engineer

### 목표
요청된 태스크에서 구현(생성/수정/리팩토링)을 수행한다.

### 입력
- task workflow step
- read/write 대상 파일 경로
- 제한된 컨텍스트 스냅샷

### 제약
- 기존 동작을 깨는 대규모 구조 변경은 최소화
- 허용되지 않은 파일 삭제/영구 데이터 변경은 사전 승인 필요

### 출력
- 변경된 파일 목록
- 실행 결과 diff/metadata append

### 성공기준
- 런타임 오류 없이 파일이 저장됨
- `metadata.fileChanges`가 실행 내용과 정합

## Agent Contract: qa

### 목표
동작/결과 검증 및 테스트 신호를 남긴다.

### 입력
- 타스크 상태 및 현재 fileChanges
- 실행 로그/테스트 커맨드 문맥

### 제약
- 재현 가능한 조건에서만 검증 수행

### 출력
- pass/fail 요약
- 실패 시 개선 제안

### 성공기준
- 실패 원인과 다음 조치 제안이 구조화되어 존재

## Skill Contract: analyze_error_logs

### 목표
실행 실패 로그로 원인을 정제하고 조치 방향을 제시한다.

### 입력
- 에러 로그 스트링
- 최근 workflow step/상태

### 제약
- 재귀 호출로 무한 루프 방지(최대 재시도 기준 준수)

### 출력
- 원인 가설 및 수정 제안

### 성공기준
- 조치 제안이 실행 가능한 step 형태로 제시됨

## Skill Contract: write_code

### 목표
주어진 요구를 코드로 반영하고 변경 내용을 파일 단위로 반영한다.

### 입력
- 경로/범위/코드 제약
- 기존 코드 스냅샷(가능한 경우)

### 제약
- `use client` 규칙과 SSR 경계를 존중
- 경로는 프로젝트 라우터 구조(`app`/`pages`)에 맞추고, 요청이 신규 기능 페이지 생성일 경우 root 페이지를 임의로 덮어쓰지 않음
- 신규 페이지 경로가 애매하면 `ProjectProfiler` 경로 정책(`routerBase`, `pageCandidates`)에 맞춰 비루트 라우트로 유도
- UI 라이브러리 import는 실제로 존재하는 파일/내보내기 기준으로만 허용 (예: `@/components/ui/*`), 미존재 컴포넌트 import는 실패 처리
- 쓰기 실패(임포트 미존재, 경로 정합성 위반, RSC 경계 위반)는 즉시 실패로 반환하고, 메타데이터 재시도/복구 절차로 전달
- 토큰/리소스 한계 내에서 최소 변경 구현

### 출력
- 저장 파일 경로 + 적용 결과 + diff

### 성공기준
- 문법 오류 없이 저장됨
- 후속 단계에서 readback이 실패하지 않음
