# 프롬프트 계약 표준 템플릿

모든 기능 문서는 아래 템플릿을 기준으로 정리해, AI가 조건을 놓치지 않도록 합니다.

```markdown
## [항목명]

### 목표
- ...

### 입력
- 필수 입력:
- 선택 입력:

### 제약
- 상태 제약:
- 보안/안전 제약:
- 의존성 제약:

### 출력
- 출력 형식:
- 필수 항목:
- 부수 산출물(로그/메타):

### 성공기준
- 정량 기준:
- 정성 기준:
- 실패 시 처리:
```

운영 규칙
- 모든 문서 상단에 태그를 둔다. (예: `#agent`, `#api`, `#workflow`, `#safety`)
- 핵심 행위는 1개 항목 1개 계약 단위로 작성한다.
- 상태 의존 로직은 `전제 조건`과 `사후 조건`을 반드시 분리한다.
- 엔드포인트/스킬 변경 시 본 템플릿 항목만 수정하고 핵심 문서는 참조 관계만 갱신한다.
